# Альфа-старт для предпринимателей

Кликабельный демо-прототип раздела экосистемы **Альфа Онлайн для бизнеса** с реально
работающим ИИ-ядром (DeepSeek). Проводит предпринимателя от идеи до первых продаж:
оценка идеи и кредита → регистрация → стартовый пакет → витрина → CRM → тендеры →
аналитика, плюс бонусный «Финансовый радар» и кинематографичный демо-сценарий.

> Демо-прототип, не финансовый продукт. Все прогнозы — аналитические сценарии,
> не инвестиционная и не кредитная рекомендация.

## Быстрый старт

```bash
npm install
cp .env.example .env      # при необходимости впишите DEEPSEEK_API_KEY
npm run dev               # фронт (5173) + прокси (8787) одновременно
```

- Фронт: http://localhost:5173 · Прокси: http://localhost:8787 (health: `/api/health`)
- Без `DEEPSEEK_API_KEY` LIVE-режим корректно откатывается на детерминированную
  модель (`src/lib/evaluateIdea.ts`) — приложение полностью работает офлайн.

| Команда           | Что делает                           |
| ----------------- | ------------------------------------ |
| `npm run dev`     | фронт + прокси вместе (concurrently) |
| `npm run dev:web` | только Vite                          |
| `npm run dev:api` | только Express-прокси                |
| `npm run build`   | типизация + прод-сборка              |
| `npm run typecheck` | только типизация                   |

## Стек

React 18 · Vite · TypeScript · Tailwind · React Router · Recharts · framer-motion ·
lucide-react · zustand · zod. Лёгкий бэкенд-прокси на Node/Express для DeepSeek.

## Два режима

| Режим     | Источник данных | Когда использовать |
| --------- | --------------- | ------------------ |
| **ДЕМО**  | «Золотые» данные `src/mock` — офлайн, воспроизводимо | Основной для показа |
| **LIVE**  | `POST /api/analyze` → DeepSeek (ключ только на сервере) + zod-валидация + детерминированный фоллбэк | Доказательство «работает по-настоящему» |

Тумблер — в шапке; дефолт из `VITE_DEMO_MODE`. Режим влияет на источник анализа в
Блоке 1; интерактив остальных блоков (витрина, CRM, тендеры…) работает на локальной
детерминированной логике всегда. Демо-сценарий `/demo-story` офлайн в любом режиме.

## Сценарий демо по шагам (golden path)

Персона: **Анна, 24 года** — открывает кофейню в спальном районе Казани.
400 000 ₽ своих, думает о кредите.

1. **Онбординг** `/onboarding` — маршрут из 8 шагов, «Начать».
2. **Дашборд** `/dashboard` — хаб: путь, «следующее действие», плитки с цифрами
   (закрытые блоки видны как тизер с замком).
3. **Оценка идеи** `/idea` — кнопка «Автозаполнить» вернёт пресет персоны.
   «Оценить идею» → «живой» расчёт → вердикт, 3 сценария, метрики, рынок и
   **безопасная сумма кредита** со светофором и стресс-тестом (спрос −30%).
4. **Регистрация** `/registration` — мастер: рекомендация формы (ИП) → данные →
   ОКВЭД → мок-подпись → «Бизнес зарегистрирован».
5. **Стартовый пакет** (там же) — карточки-тумблеры «до 25 лет». «Активировать всё»
   мгновенно открывает витрину, CRM, аналитику, тендеры и радар (сайдбар/дашборд).
6. **Локальная витрина** `/storefront` — автокарточка бизнеса, настройка аудитории,
   вид «как видит клиент», метрики охвата.
7. **Администратор в кармане** `/admin` — записи → «запросить оплату» → QR/ссылка →
   «оплачено» → сводка «Деньги» с налоговой подсказкой.
8. **Тендеры** `/tenders` — подбор с матч-скором, детали лота, чек-лист документов,
   льготная гарантия для новичка до 25 (калькулятор + мастер выпуска).
9. **Аналитика МП** `/analytics` — чистая прибыль по SKU, прогноз оборота,
   рекомендации по ассортименту.
10. **Финансовый радар** `/radar` — бонус: вторая персона «Автосервис „Драйв“»,
    аномалии «расход↔выручка», прогноз кассового разрыва, овердрафт под сделку.
11. **Демо-сценарий** `/demo-story` — флагманский показ (см. ниже), ▶ и на весь экран.

**Сброс:** кнопка **«Демо заново»** в шапке возвращает всё к пресету персоны
(прогресс, регистрация, активация, витрина, записи/платежи, документы тендеров) —
показ можно повторять без перезагрузки. На формах есть «Автозаполнить».

## Что настоящее, а что мок — и почему

- **Настоящее:** вызов DeepSeek в LIVE (через серверный прокси, ключ не попадает в
  браузер), вся детерминированная математика (оценка идеи и кредита, юнит-экономика,
  матч-скор тендеров, гарантия, кассовый календарь), интерактив и персистентность.
- **Мок (осознанно):** банковские API, БКИ, маркетплейсы, госзакупки, карта района,
  QR (визуальная матрица, не сканируется), регистрация/ЭЦП. Реалистичные мок-данные
  (`src/mock/*`) подаются в промпт DeepSeek как «контекст рынка» — демо не зависит
  от внешних систем и воспроизводимо на показе.
- **Согласованность цифр:** дашборд считает ключевые цифры теми же функциями и
  сторами, что и сами блоки (`features/dashboard/useBlockWidgets.ts`) — числа
  совпадают по построению. Исключение — `/demo-story`: там курированный «золотой
  слепок» (`src/mock/goldenAnalysis.ts`) под драматургию показа.

## Путь и активация

Единый путь из 8 шагов (`lib/journey.ts`): онбординг → идея → регистрация → пакет →
витрина → CRM → тендеры → аналитика. Блоки 2–6 закрыты до активации в стартовом
пакете (`useActivationStore` + `GatedRoute`): тумблер «включает» раздел в сайдбаре,
на дашборде и в роутере мгновенно. Прогресс и активация — в `localStorage`.

## Флагманский демо-сценарий (собран на Claude Fable 5)

`/demo-story` — кинематографичная история «Кофейни Анны» из 6 сцен (~117 сек в
авто-режиме): завязка-страх → ввод идеи и «живой» ИИ-анализ → **кульминация: три
сценария с рисующимися графиками, точками окупаемости и зелёным светофором
кредита** → плашка-инсайт «безопасно 450 000 ₽, окупаемость 7 мес даже при
спросе −30%» → монтаж пути за 25 сек → финал с CTA.

Полностью офлайн и детерминированно (`src/mock/goldenAnalysis.ts`). Управление:
пульт (Сначала · Назад · Играть/Пауза · Далее + кликабельный прогресс), горячие
клавиши: Пробел, ←/→, R — сначала, F — на весь экран (для проектора). Размеры
кадра в `cqw` — одинаково читается в окне и в фуллскрине. Архитектура:
`director.ts` (zustand-режиссёр), `scenes/` (реестр: новая сцена = одна строка),
`bits/` (Counter, TypeWriter, DistrictMap, MiniPaybackChart, CreditLight).
